Following are changes of SDK code made by GuangZhou team in recenty 2 mouths：

1. 用openssl做非对称加解密，在请求中加了sign，服务器用它来认证我们。
using openssl library to encryt sign and take it in Ad Request, server will authenticate SDK client through this "sign" parameter.
This change increases SDK file size from xx ks to 2M.

2. ad请求参数由json变为放在web form里面，这样nginx服务器可以直接读取，不用解析json
change request parameters to web form format instead of a json body
this allow nginx server get parameter directlly, no need to parse json body

3. 增加了user id和user group，用作SDK client的区分和分组
adding user id and user group parameters in AD request, to identify clients and randomly divied them to 100 groups for A/B test

4. 中国地区增加imei, androidid的上报
for china mainland users, adding imei, androidid parameters in AD request

5. 增加application id概念，表示一个开发者（publisher）下面的一个app
adding application id, while a application is a APP belongs to a publisher

6. 增加了全屏广告和视频广告的倒计时
a count down timers will show in top right of screen when showing a intersitial AD or video AD in full screen.
It tells user how many seconds reminds this Ad's show

7. 和服务器的通信由http改为https
Change http to https, for ad request, impression request and click request
